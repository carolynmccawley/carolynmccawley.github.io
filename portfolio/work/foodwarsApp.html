Food Wars App

React Native App

Built With:
  • React Native
  • React Native Paper
  • Yelp Fusion API
  • Google Maps

The Food Wars app is a collaboration between Carolyn McCawley and Sam Ko. The app is marketed towards millenials who live in cites who have gone through the struggle of trying to pick a restaurant amongst friends. Carolyn was the software developer and was in charge of building out the functinality of the app. The app has three different 'quizzes' for the different meals and based on the quiz results, the users' preferences, and their location, a restaurant is generated using the Yelp API.The user is then shown the menu and directions to the restaurant.

Logic

The first step in building the app was determining how a restaurant would be chosen. We knew that the Yelp API would be implemented to generate a result, but the difficult decision was determining which category or restaurant type that the user wanted. We decided to break the decision into the three types of meals: Breakfast, Lunch & Dinner, and Dessert. We approached the vast number of categories by narrowing it down using a survey to determine people's favorite foods and then created a bracket where the user narrows it down to their favorite dish. After the user determines their desired food, they are given a restaurant that reflects that favorite food.

Navigation

After determining how a category would be selected, the next hurdle was creating a user flow and navigation within the app. Carolyn built a navigation bar at the bottom of the screen that enabled the user to redirect themselves to their saved restaurants, the home page or the logout screen. The homepage displays the three types of quizzes: Breakfast, Lunch & Dinner, and Dessert. Each quiz redirects the user to a distinct ‘quiz’ that aids in discovering which type of food they desire.

Functionality

After shaping the navigation with a stack navigator, Carolyn worked on the functionality; this included saving the user’s inputs within the local state and store for later access. The state stored the user’s conditions (such as radius and price point), their location and the results from the Yelp API. Information such as user authorization and saved restaurants are passed into the Firebase Database.

Building Out

Once the basics were there, Carolyn began to add in the extra functions. This included calling the Yelp API in order to show reviews based on the results of the initial call. Upon seeing the results, the user can view more, get directions to the restaurant, save to their collection, and get another option using the shuffle button.

Integrating UX

After polishing the functionality and testing, Carolyn took Sam’s Figma sketches and created stylesheets to reflect the design of the app. This included assigning a unique color to each quiz, creating custom buttons, and displaying distinct illustrations for the different foods and screens.

Refining

The final steps consisted of user testing to ensure there were no bugs within the app and refining the visuals of the app. Afterwards, the app was deployed on Expo.
